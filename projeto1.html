Definição



A modelagem dos dados foi realizada utilizando a ferramenta BrModelo, solução open source baseada em java, na versão on premisse.

Requisitos

Representar as entidades e os relacionamentos do seguinte cenário

    Tópico 1

  • Os livros são escritos por pelo menos um autor. Um autor pode escrever vários livros.

    • Tópico 2

  • Do livro deverá ser armazenado o título, o código ISBN, o número e o ano da edição.

    • Tópico 3

  • Dos autores deve-se armazenar: um código, nome, data de nascimento, país de origem, titulação

    • Tópico 4

  • Um livro é publicado por apenas uma editora, da qual deve ser armazenado: nome, cidade, UF, e-mail e todos os telefones de contato.

Modelagem

Relacionamento entre as tabelas

O processo de modelagem deste projeto foi feito inteiramente no BrModelo. Após analisar os requisitos propostos, busquei identificar além dos campos necessários, a cardinalidade para cada relacionamento. Entre a tabela de Livro e Autor, a cardinalidade, é de 1 para muitos (1:n), significando dizer que um livro pode ser escrito por um ou vários autores, já entre a tabela de Autor e Livro, a cardinalidade, é de 0 para muitos (0:n), indicando que um autor pode escrever nenhum ou vários livros.

É possível notar que tabela fato só se relaciona com tabela dimensão e vice-versa. O relacionamento entre as tabelas é de um para muitos (1:*), com o filtro sempre indo da dimensão para a fato.

Neste projeto não temos tabela ponte (fato sem fato) nem diferença de granularidade que poderia gerar relacionamento muitos para muitos (*:*).
