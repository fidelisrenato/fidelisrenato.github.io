Link do Relatório

Este relatório foi desenvolvido, inteiramente com recursos nativos do Power BI, para atender uma demanda pontual, utilizando uma base de dados real, porém alterada para proteger os dados da fonte.

Requisitos

O que foi pedido

    Página única

  • Deve existir 3 segmentadores, Ano, Mês e Tipo Unidade para as métricas “Ocorrências Ano Anterior" e "Ocorrências Ano Atual", que permitirá ao gestor escolher por qual(is) segmentadores ele desejar analisar o relatório;
  • 2 cards que exibam o total de Ocorrências do ano anterior e do ano atual;
  • Um visual com Total de Ocorrências por distrito;
  • Um visual com Total de Ocorrências por tipo;
  • Um visual com o percentual de Ocorrências por tipo do ano anterior;
  • Um visual com o percentual de Ocorrências por tipo do ano atual;
Definições da área
  • Ocorrências de Conectividade:

    • ∑ (Total de Ocorrências)

Análise de Dados

Extraindo informações

Por se tratar de uma base já conhecida a análise exploratória, foi realizada de maneira rápida, porém com atenção aos requisitos estabelecidos, utilizando tabelas e expressões DAX, para conhecer os dados..

Medida do Total de Ocorrências do ano anterior:

Ocorrencias Ano Anterior = 
                    VAR EMBRANCO =             
                        CALCULATE(
                            [Ocorrências Ano Atual],
                            DATEADD(dCalendario[Data], -1, YEAR)
                     )
                    return
                    IF(
                        ISBLANK(
                            EmBranco),
                            0,
                            EmBranco
                    )

Medida do Total de Ocorrências do ano antual:

                    Ocorrencias Ano Atual =
                        VAR EMBRANCO = 
                            CALCULATE(
                                COUNT(Ocorrencias_Conectividade[CodUnidade])
                            )
                    return
                    IF(
                        ISBLANK(
                            EmBranco),
                                0,
                            EmBranco
                      )

ETL e Modelagem

Relacionamento entre as tabelas

O processo de ETL deste projeto foi feito inteiramente em Power Query por se tratar de uma base cujo volume dados é bem resumido. Também foi criada, em linguagem M, uma tabela dimensão calendário, que possibilita centralizar as datas, permitindo melhor consistência nas análises ao trabalhar com funções de inteligência de tempo.
