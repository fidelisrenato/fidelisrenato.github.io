Definição


Exercício proposto durante a mentoria, após as aulas de modelagem.


Requisitos

    Tópicos


  • Os filmes devem ser locados por clientes previamente cadastrados.

  • O cliente pode locar quantos filmes quiser.

  • O filme pertence a uma determinada categoria (Drama, Ação, Aventura, etc.)

  • Os filmes são gravados em mídias que possuem um código de identificação.

  • Um filme pode ser gravado em várias mídias, mas uma mídia só pode conter um filme.

  • Cada mídia pode estar locada para no máximo uma pessoa em um determinado tempo, mas uma pessoa pode alugar diversas mídias.

  • Em uma locação pode haver várias mídias assim como uma mídia pode estar em várias locações.

    Campos


  • Para este exemplo não há campos a serem considerados.

Modelagem

Conforme proposto na definição, este DER possui apenas Entidades com seus respectivos relacionamentos, bem como, as cardinalidades mínima e máxima para cada relacionamento.

O processo de modelagem deste projeto foi realizado utilizando a ferramenta BrModelo solução open source baseada em java, na versão on premisse. Após analisar os requisitos propostos, foi realizado o mapeamento das tabelas, bem como, definida a cardinalidade para cada relacionamento.

Entre as tabelas Cliente e Locação, foram geradas as seguintes cardinalidades: 0 para n (0:n), significando dizer que um cliente pode realizar nenhuma ou várias locações e 1 para 1 (1:1), indicando que uma locação pertence exclusivamente a um cliente.

Entre as tabelas Locação e Midia, foram geradas as seguintes cardinalidades: 1 para n (1:n), significando dizer que uma locação pode conter uma ou várias mídias e 0 para muitos (0:n), indicando que uma mídia pode estar em nenhuma ou várias locações.

Entre as tabelas Mídia e Filme, foram geradas as seguintes cardinalidades: 1 para n (1:1), significando dizer que uma mídia conterá um único filme e 0 para muitos (0:n), indicando que um filme pode ser gravado em nenhuma ou várias mídias.

Entre as tabelas Filme e Categoria, foram geradas as seguintes cardinalidades: 1 para n (1:1), significando dizer que um filme pertence a uma única categoria e 0 para muitos (0:n), indicando que uma categoria pode ter nenhum ou vários filmes.
